Git Providers — ソフトウェアフォージやソースコード管理システム（SCM）とも呼ばれる — は、Git を使用してコードベースをホストし、共同作業を行うためのツールセットを提供します。

Daytona では、お好みの Git Provider を統合することで、開発ワークフローを強化できます。この統合により、Daytona 内でリポジトリに直接アクセスし、管理することができ、既存のプロジェクトからのワークスペース作成が簡単になります。

Git Provider を追加することで、認証情報がワークスペースに安全に埋め込まれます。これにより、認証を繰り返すことなく、シームレスな Git 操作が可能になります。

## Git Providerの追加

Daytonaでは、Git Providerを追加するオプションを提供しており、お好みのGit Providerに接続してDaytonaから直接リポジトリにアクセスできます。

1. 以下のコマンドを実行してGit Providerを追加します：

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytonaは追加したいGit Providerを選択するよう促します。

```text
Choose a Git provider
GitHub
GitLab
Bitbucket
Other
```

2. リストから設定したいGit Providerを選択します：

- __[GitHub](#github)__

    GitHubでリポジトリをホストしている場合に選択します。

- __[GitLab](#gitlab)__

    GitLabでリポジトリをホストしている場合に選択します。

- __[Bitbucket](#bitbucket)__

    AtlassianのBitbucket Cloudプラットフォームでリポジトリをホストしている場合に選択します。

- __[GitHub Enterprise Server](#github-enterprise-server)__

    GitHub Enterprise Serverでリポジトリをホストしている場合に選択します。

- __[GitLab Self-Managed](#gitlab-self-managed)__

    GitLabのセルフホスト型プラットフォームでリポジトリをホストしている場合に選択します。

    セルフホスト型のGitLab Community Edition（CE）インストール、GitLab Self-Managed、およびGitLab Dedicatedプラットフォームがサポートされています。

- __[Bitbucket Server](#bitbucket-server)__

    Bitbucket Serverでリポジトリをホストしている場合に選択します。

- __[Codeberg](#codeberg)__

    Codebergでリポジトリをホストしている場合に選択します。

- __[Gitea](#gitea)__

    Giteaでリポジトリをホストしている場合に選択します。

- __[Gitness](#gitness)__

    Gitnessでリポジトリをホストしている場合に選択します。

- __[Azure DevOps](#azure-devops)__

    Azure DevOpsでリポジトリをホストしている場合に選択します。

- __[AWS CodeCommit](#aws-codecommit)__

    AWS CodeCommitでリポジトリをホストしている場合に選択します。

- __[Gogs](#gogs)__

    Gogsでリポジトリをホストしている場合に選択します。

- __[Gitee](#gitee)__

    Giteeでリポジトリをホストしている場合に選択します。

3. Daytonaが認証を行い、リポジトリへの安全なアクセスを有効にするために、Git Providerの認証情報を入力します。

```text
Personal access token
>
```

一部のGit Providerでは、設定を完了するために_ユーザー名_やセルフマネージドAPIインスタンスの_URL_などの追加情報が必要です。

4. Daytona内でGit Providerを識別するためのエイリアスを入力します。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

認証が成功すると、Daytonaは以下のメッセージを表示します：

```text
Git provider has been registered
```

## 接続されたGitプロバイダーの一覧表示

Daytonaは、以前に作成されたすべてのGitプロバイダーを一覧表示することで、Gitプロバイダーを追跡するオプションを提供します。

1. 現在接続されているGitプロバイダーを一覧表示するには、次のコマンドを実行します：

```shell
daytona git-providers list
```

このコマンドを実行すると、Daytonaは接続されているGitプロバイダーの一覧を表示します。

```text
GitHub (alias)
GitLab (alias)
```

## Git Providerの削除

Daytonaは、Git Providerを削除するオプションを提供し、不要になったものを管理・削除するのに役立ちます。Git Providerが削除されると、別のGit Providerを追加しない限り、Workspaceを管理またはデプロイすることができなくなります。

1. 以下のコマンドを実行してGit Providerを削除します：

```shell
daytona git-providers delete
```

このコマンドを実行すると、Daytonaは削除可能な既存のGit Providerのリストを表示します。

```text
Choose a Git provider
GitHub
GitLab
```

2. 削除したいGit Providerを選択します。

選択が正常に完了すると、Daytonaは以下のメッセージを表示します：

```text
Git provider has been removed
```

## Gogs

DaytonaはGogsアカウントに接続し、Daytonaから直接リポジトリにアクセスするオプションを提供します。

1. 次のコマンドを実行してGogsをGit Providerとして追加します：

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytonaは追加したいGit Providerを選択するよう促します。リストから`Other`を選択し、次に`Gogs`を選択します。

```text
Choose a Git provider
> Other
```

```text
Choose a Git provider
> Gogs
```

GogsをGit Providerとして選択した後、Self-Managed API URL（GogsインストールのAPIのベースURL）とPersonal Access Tokenの入力を求められます。

Personal Access Tokenを作成するには、以下の手順に従ってください：

1. Gogsにログインし、プロフィールアバターをクリックしてユーザーメニューを開きます。
2. ユーザーメニューで__`Your Settings`__をクリックします。リポジトリへの管理者アクセス権があることを確認してください。
3. サイドバーの__`Applications`__に移動します。
4. Manage personal access tokensセクションで、__`Generate New Token`__をクリックします。
5. トークンの名前を入力し、__`Generate Token`__をクリックします。
6. トークンをコピーして安全に保存します。

<Aside type="note">
Gogsは細かい粒度のトークンをサポートしていません。各Personal Access Tokenはあなたのアカウントへの完全なアクセス権を持ちます
</Aside>

## Gitee

DaytonaはGiteeアカウントを接続し、Daytonaから直接リポジトリにアクセスするオプションを提供します。

1. 次のコマンドを実行してGiteeをGit Providerとして追加します：

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytonaは追加したいGit Providerを選択するよう促します。リストから`Other`を選択し、次に`Gitee`を選択します。

```text
Choose a Git provider
> Other
```

```text
Choose a Git provider
> Gitee
```

2. Daytonaが認証を行い、Giteeリポジトリへの安全なアクセスを有効にするために、Personal Access Tokenを入力します。

Personal Access Tokenを作成するには、以下の手順に従ってください：

1. Giteeにログインし、プロフィールアバターをクリックしてユーザーメニューを開きます。
2. ユーザーメニューで__`Settings`__をクリックします。リポジトリへの管理者アクセス権があることを確認してください。
3. サイドバーの__`Personal access tokens`__に移動します。
4. Manage personal access tokensセクションで、Generate New Tokenをクリックします。
5. トークンの説明を入力し、以下のスコープを選択します：
    - __`user_info`__
    - __`projects`__
    - __`pull_requests`__
    - __`groups`__
    - __`emails`__
6. __`Commit`__をクリックしてトークンを生成します。確認のためにパスワードの入力を求められます。
7. トークンをコピーして安全に保存します。

```text
Personal access token
>
```

3. Daytona内でGit Providerを識別するためのGit Provider aliasを入力します。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

認証が成功すると、Daytonaは以下のメッセージを表示します：

```text
Git provider has been registered
```
