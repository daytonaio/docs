Git プロバイダー（ソフトウェアフォージやソースコードマネージャー（SCM）とも呼ばれます）は、Git を使ってコードベースをホストし、共同作業を行うためのツールセットを提供します。

Daytona では、お好みの Git プロバイダーと統合することができ、開発ワークフローを強化します。この統合により、Daytona 内でリポジトリへ直接アクセス・管理でき、既存プロジェクトからのワークスペース作成が簡単になります。

Git プロバイダーを追加すると、あなたの認証情報が安全にワークスペースへ組み込まれます。これにより、繰り返し認証することなく、シームレスに Git 操作を行うことができます。

## Gitプロバイダーを追加する

Daytona では、Gitプロバイダーを追加するオプションが用意されており、お好みのGitプロバイダーと接続して、リポジトリに直接アクセスすることができます。

1. 次のコマンドを実行してGitプロバイダーを追加します:

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytonaが追加したいGitプロバイダーの選択を促します。

```text
Gitプロバイダーを選択してください
GitHub
GitLab
Bitbucket
その他
```

2. リストから設定したいGitプロバイダーを選択します:

- __[GitHub](#github)__

    リポジトリをGitHubでホストしている場合に選択してください。

- __[GitLab](#gitlab)__

    リポジトリをGitLabでホストしている場合に選択してください。

- __[Bitbucket](#bitbucket)__

    リポジトリをAtlassianのBitbucket Cloudプラットフォームでホストしている場合に選択してください。

- __[GitHub Enterprise Server](#github-enterprise-server)__

    リポジトリをGitHub Enterprise Serverでホストしている場合に選択してください。

- __[GitLab Self-Managed](#gitlab-self-managed)__

    リポジトリをGitLabのセルフホスト型プラットフォームでホストしている場合に選択してください。

    セルフホスト型のGitLab Community Edition (CE) インストール、GitLab Self-Managed、およびGitLab Dedicatedプラットフォームがサポートされています。

- __[Bitbucket Server](#bitbucket-server)__

    リポジトリをBitbucket Serverでホストしている場合に選択してください。

- __[Codeberg](#codeberg)__

    リポジトリをCodebergでホストしている場合に選択してください。

- __[Gitea](#gitea)__

    リポジトリをGiteaでホストしている場合に選択してください。

- __[Gitness](#gitness)__

    リポジトリをGitnessでホストしている場合に選択してください。

- __[Azure DevOps](#azure-devops)__

    リポジトリをAzure DevOpsでホストしている場合に選択してください。

- __[AWS CodeCommit](#aws-codecommit)__

    リポジトリをAWS CodeCommitでホストしている場合に選択してください。

- __[Gogs](#gogs)__

    リポジトリをGogsでホストしている場合に選択してください。

- __[Gitee](#gitee)__

    リポジトリをGiteeでホストしている場合に選択してください。

3. Daytonaが認証し、リポジトリへの安全なアクセスを有効にするために、Gitプロバイダーの認証情報を入力してください。

```text
パーソナルアクセストークン
>
```

一部のGitプロバイダーでは、設定を完了するために_ユーザー名_やセルフマネージドAPIインスタンスの_URL_など、追加情報の入力が必要な場合があります。

4. Daytona内で識別するためのGitプロバイダーのエイリアスを入力してください。

```text
エイリアス                                                                         
空欄の場合はユーザー名がデフォルトになります                                        
>
```

認証が成功すると、Daytonaは次のメッセージを表示します:

```text
Gitプロバイダーが登録されました
```

## 接続済みのGitプロバイダーを一覧表示

Daytonaは、これまでに作成したGitプロバイダーを一覧表示することで、Gitプロバイダーを管理するオプションを提供します。

1. 現在接続されているGitプロバイダーを一覧表示するには、次のコマンドを実行します。

```shell
daytona git-providers list
```

このコマンドを実行すると、Daytonaは接続済みのGitプロバイダーの一覧を表示します。

```text
GitHub (alias)
GitLab (alias)
```

## Gitプロバイダーを更新する

Daytonaは、既存のGitプロバイダーを更新するオプションを提供しており、最新の機能強化やバグ修正を適用できます。

1. 次のコマンドを実行してGitプロバイダーを更新します。

```shell
daytona git-providers update
```

このコマンドを実行すると、Daytonaは接続されているGitプロバイダーの一覧を表示します。

```text
Choose a Git provider 
> github (alias)
```

2. 一覧から更新したいGitプロバイダーを選択します。

3. Daytonaが認証し、リポジトリへの安全なアクセスを有効にするために、Gitプロバイダーの認証情報を入力します。

```text
Personal access token
>
```

一部のGitプロバイダーでは、設定を完了するために_ユーザー名_やセルフマネージドAPIインスタンスの_URL_など、追加情報の入力が必要です。

4. Daytona内で識別するために、Gitプロバイダーのエイリアスを入力します。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

更新が成功すると、Daytonaは次のメッセージを表示します。

```text
Git provider has been updated
```

## Gitプロバイダーの削除

Daytona では、不要になった Gitプロバイダーを管理・削除するためのオプションが用意されています。Gitプロバイダーを削除すると、別の Gitプロバイダーを追加しない限り、Workspace の管理やデプロイができなくなります。

1. Gitプロバイダーを削除するには、次のコマンドを実行します。

```shell
daytona git-providers delete
```

このコマンドを実行すると、Daytona は削除可能な既存の Gitプロバイダーの一覧を表示します。

```text
Gitプロバイダーを選択してください
GitHub
GitLab
```

2. 削除したい Gitプロバイダーを選択します。

正常に選択されると、Daytona は次のメッセージを表示します。

```text
Gitプロバイダーが削除されました
```

## GitHub

Daytona は、GitHub アカウントを接続し、リポジトリに直接アクセスできるオプションを提供しています。

1. 次のコマンドを実行して、GitHub を Git Provider として追加します。

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytona から追加したい Git Provider の選択を求められます。リストから `GitHub` を選択してください。

```text
Choose a Git provider
> GitHub
```

GitHub を Git Provider として選択した後、Personal Access Token の入力を求められます。

Personal Access Token の作成方法については、GitHub の [Creating a personal access token (classic)][gh-token] ドキュメントをご覧ください。

2. Daytona で認証し、GitHub リポジトリへの安全なアクセスを有効にするために、Personal Access Token を入力してください。

```text
Personal access token
>
```

3. Daytona 内で識別するための Git Provider のエイリアスを入力してください。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

認証が成功すると、Daytona は次のメッセージを表示します。

```text
Git provider has been registered
```

</Aside>

2. Daytona で認証し、AWS CodeCommit リポジトリへの安全なアクセスを有効にするために、ユーザー名、セルフマネージド API URL、およびパーソナルアクセストークンを入力してください。

## Gogs

Daytona は、Gogs アカウントを接続し、リポジトリへ直接アクセスできるオプションを提供しています。

1. 次のコマンドを実行して、Gogs を Git プロバイダーとして追加します:

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytona から追加したい Git プロバイダーの選択を求められます。リストから `Other` を選択し、その後 `Gogs` を選択してください。

```text
Choose a Git provider
> Other
```

```text
Choose a Git provider
> Gogs
```

Gogs を Git プロバイダーとして選択した後、Self-Managed API URL（Gogs インストールの API のベース URL）と Personal Access Token の入力を求められます。

Personal Access Token を作成するには、以下の手順に従ってください:

1. Gogs にログインし、プロフィールアバターをクリックしてユーザーメニューを開きます。
2. ユーザーメニューから __`Your Settings`__ をクリックします。リポジトリへの管理者権限があることを確認してください。
3. サイドバーの __`Applications`__ に移動します。
4. 「Manage personal access tokens」セクションで __`Generate New Token`__ をクリックします。
5. トークンの名前を入力し、__`Generate Token`__ をクリックします。
6. トークンをコピーして安全な場所に保存してください。

<Aside type="note">
Gogs は細かい権限設定ができるトークンをサポートしていません。各 Personal Access Token はアカウントへのフルアクセス権を持ちます。
</Aside>

<br />

2. Self-Managed API URL と Personal Access Token を入力し、Daytona が認証して Gogs リポジトリへの安全なアクセスを有効にします。

```text
Self-managed API URL
For example: https://gogs-host.com
>
```

```text
Personal access token
>
```

3. Daytona 内で識別するための Git プロバイダーのエイリアスを入力します。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

認証が成功すると、Daytona は次のメッセージを表示します:

```text
Git provider has been registered
```

## Gitee

Daytona は、Gitee アカウントを接続し、リポジトリに直接アクセスできるオプションを提供しています。

1. 次のコマンドを実行して、Gitee を Git プロバイダーとして追加します。

```shell
daytona git-providers add
```

このコマンドを実行すると、Daytona から追加したい Git プロバイダーの選択を求められます。リストから `Other` を選択し、その後 `Gitee` を選択してください。

```text
Choose a Git provider
> Other
```

```text
Choose a Git provider
> Gitee
```

2. Daytona が認証し、Gitee リポジトリへの安全なアクセスを有効にするために、Personal Access Token を入力します。

Personal Access Token を作成するには、以下の手順に従ってください。

1. Gitee にログインし、プロフィールアバターをクリックしてユーザーメニューを開きます。
2. ユーザーメニューで __`Settings`__ をクリックします。リポジトリへの管理者権限があることを確認してください。
3. サイドバーの __`Personal access tokens`__ に移動します。
4. Manage personal access tokens セクションで、Generate New Token をクリックします。
5. トークンの説明を入力し、次のスコープを選択します:
    - __`user_info`__
    - __`projects`__
    - __`pull_requests`__
    - __`groups`__
    - __`emails`__
6. __`Commit`__ をクリックしてトークンを生成します。確認のためパスワードの入力を求められます。
7. トークンをコピーして安全に保存してください。

```text
Personal access token
>
```

3. Daytona 内で識別するために、Git プロバイダーのエイリアスを入力します。

```text
Alias                                                                         
Will default to username if left empty                                        
>
```

認証が成功すると、Daytona は次のメッセージを表示します。

```text
Git provider has been registered
```
