プロバイダーは、Daytonaがさまざまな技術と連携して開発環境を作成・管理するためのプラグインです。プロバイダーは基盤となる技術の複雑さを抽象化し、Daytonaがコンテナ化、オーケストレーション、またはクラウドベースの仮想マシンを通じて環境をデプロイおよび実行する際の基盤エンジンとして機能します。

Daytonaのアーキテクチャでは、プロバイダーの役割を独立したサービスとして分散化しています。[Daytona Server](/docs/ja/usage/server) は、ワークスペースの作成やライフサイクル管理に関する操作を実行するためにプロバイダーと通信します。

## プロバイダーのインストール

Daytona では、Daytona Server と連携するためのプロバイダーをインストールするオプションが用意されています。新しいプロバイダーをインストールすると、そのプロバイダーがワークスペースのデプロイとライフサイクル管理を担当します。

1. 次のコマンドを実行してプロバイダーをインストールします。

```shell
daytona provider install
```

:::note
Daytona は、プロバイダーを管理するための [Runner](/docs/ja/usage/runners) の選択を求めます。
:::

<br />

2. インストールしたいプロバイダーをリストから選択するか、特定のバージョンを選択します。

```text
インストールするプロバイダーを選択
7 件
===
Docker
v0.0.0
DigitalOcean
v0.0.0
AWS
v0.0.0
Azure
v0.0.0
GCP
v0.0.0
Hetzner
v0.0.0
Fly
v0.0.0

特定のバージョンを選択
```

プロバイダーを選択すると、Daytona が選択したプロバイダーをインストールおよび設定し、ワークスペースの管理とデプロイに利用できるようになります。これで、作成したプロバイダーと一緒に使用するための [Target](/docs/ja/usage/targets) を作成できます。

```text
プロバイダー <provider> のインストールが正常に完了しました
```

## プロバイダーの一覧

Daytona では、インストールされているすべてのプロバイダーを一覧表示するオプションが用意されており、ワークスペース管理に利用可能なプロバイダーの概要を確認できます。

1. 次のコマンドを実行して、インストールされているすべてのプロバイダーを一覧表示します。

```shell
daytona provider list
```

このコマンドを実行すると、Daytona はインストール済みプロバイダーとそれぞれのバージョンを一覧表示します。

```text
Name         Runner         Name               Version               
──────────────────────────────────────────────────────
Docker       local          docker-provider    v0.0.0
```

## プロバイダーを更新する

Daytona では、既存のプロバイダーを更新するオプションが用意されており、最新の機能強化やバグ修正を適用できます。

1. 次のコマンドを実行してプロバイダーを更新します。

```shell
daytona provider update
```

2. プロバイダーを管理する Runner を選択します。

```text
Choose a Runner To Manage Providers
1 item
===
local
local
```

3. 更新したいプロバイダーをリストから選択します。

```text
Choose a provider to update
1 item
===
Docker (Runner <runner>)
v0.0.0
```

```text
Provider docker-provider has been successfully updated
```

## プロバイダーのアンインストール

Daytona では、既存のプロバイダーをアンインストールするオプションが用意されており、不要になったプロバイダーを削除して管理することができます。

1. 次のコマンドを実行してプロバイダーをアンインストールします。

```shell
daytona provider uninstall
```

2. プロバイダーを管理する Runner を選択します。

```text
Choose a Runner To Manage Providers
1 item
===
local
local
```

3. アンインストールしたいプロバイダーをリストから選択します。

```text
Choose a provider to uninstall
1 item
===
Docker (Runner <runner>)
v0.0.0
```

```text
Provider docker-provider has been successfully uninstalled
```

## Docker

Dockerプロバイダーを使用すると、DaytonaはワークスペースプロジェクトをDockerコンテナとして作成できます。DaytonaはデフォルトでDocker用のプロバイダー（`docker-provider`）をインストールし、Dockerプロバイダーを使用したデフォルトの[ターゲット](/docs/ja/usage/targets)を追加します。これにより、ローカルマシン上でワークスペースを作成できるようになります。

1. 次のコマンドを実行してDockerプロバイダーをインストールします。

```shell
daytona provider install
```

2. Dockerプロバイダーを管理するRunnerを選択します。

```text
Choose a Runner To Manage Providers
1 item
===
local
local
```

3. プロバイダーとしてDockerをインストールするために**`Docker`**を選択します。

```text
Choose a provider to install
===
> Docker
  v0.0.0
```

プロバイダーを選択すると、Daytonaが選択したプロバイダーをインストールおよび設定し、ワークスペースの管理とデプロイに利用できるようにします。

```text
⡿  Installing...
```

```text
Provider docker-provider has been successfully installed
```

```text
local
```

3. **`AWS`** を選択して、プロバイダーとしてAWSをインストールします。


プロバイダーを選択すると、Daytonaが選択したプロバイダーをインストールおよび設定し、Workspaceの管理とデプロイに利用できるようになります。

## Azure

Azure Provider を使用すると、Daytona は Azure のコンピュートインスタンス上に Workspace プロジェクトを作成できます。この Provider は Daytona と Azure を統合し、スケーラブルで柔軟なクラウド環境で Workspace を管理できるようにします。

1. 次のコマンドを実行して Azure Provider をインストールします。

```shell
daytona provider install
```

2. Azure Provider を管理する Runner を選択します。

```text
Choose a Runner To Manage Providers
1 item
===
local
local
```

3. Provider として **`Azure`** をインストールするために選択します。

```text
Choose a provider to install
===
> Azure
  v0.0.0
```

Provider を選択すると、Daytona が選択した Provider をインストールおよび設定し、Workspace の管理とデプロイに利用できるようにします。

```text
⡿  Installing...
```

```text
Provider azure-provider has been successfully installed
```

:::note
Azure Provider が、該当する Azure サブスクリプションの `Contributor` ロールを持つサービスプリンシパルにリンクされていることを確認してください。
<br />
`Contributor` ロールを持つサービスプリンシパルを作成するには、次の Azure CLI コマンドを使用します。
<br />

```shell
az ad sp create-for-rbac --role Contributor --scopes /subscriptions/<subscription-id>
```

サービスプリンシパルの作成と設定の詳細な手順は、公式ドキュメント [Create an Azure service principal with Azure CLI](https://learn.microsoft.com/en-us/cli/azure/azure-cli-sp-tutorial-1?tabs=bash) をご参照ください。
:::

## GCP

GCPプロバイダーを使用すると、DaytonaはGoogle Cloud Platform（GCP）のコンピュートインスタンス上にWorkspaceプロジェクトを作成できます。このプロバイダーはDaytonaとGCPを統合し、スケーラブルで柔軟なクラウド環境でWorkspaceを管理できるようにします。

:::note
GCPプロバイダーを使用する前に、`Compute Admin`ロールを持つGCPサービスアカウントを作成してください。サービスアカウントキーをJSON形式でダウンロードし、認証のためにGCPプロバイダーに提供します。サービスアカウントの作成と設定の詳細な手順は[こちら](https://cloud.google.com/iam/docs/service-accounts-create#console)をご覧ください。
:::

<br />

```shell
daytona provider install
```

2. GCPプロバイダーを管理するRunnerを選択します。

```text
Choose a Runner To Manage Providers
1 item
===
local
local
```

3. **`GCP`** を選択して、プロバイダーとしてGCPをインストールします。

```text
Choose a provider to install
===
> GCP
  v0.0.0
```

プロバイダーを選択すると、Daytonaが選択したプロバイダーをインストールおよび設定し、Workspaceの管理とデプロイに利用できるようになります。
