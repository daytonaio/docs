コンテナレジストリは、指定したレジストリサーバーからコンテナイメージを取得するために使用される認証情報を保存します。コンテナレジストリの認証情報を追加することで、プライベートイメージやプライベートレジストリにホストされているイメージからワークスペースを作成したいユーザーにとって便利です。

コンテナレジストリは環境変数を通じて管理され、認証情報を安全に保存・管理できます。Daytona では、ワークスペース用のコンテナレジストリを設定、一覧表示、削除するためのオプションが用意されています。

Daytona では、すべてのワークスペースに適用されるグローバルな環境変数を設定する方法と、特定のワークスペース用に環境変数を設定する方法が提供されています。特定のワークスペース用に環境変数を設定するには、新しいワークスペースを作成する際に `--env` フラグを使用するか、ワークスペースのテンプレートで環境変数を設定してください。すべての新しいワークスペースに適用されるサーバー全体の環境変数を設定するには、`daytona env` コマンドを使用します。

## コンテナレジストリを設定する

Daytona では、環境変数を使って Workspace 用のコンテナレジストリを設定するオプションが用意されています。

1. 次のコマンドを実行して、コンテナレジストリの環境変数を設定します。

```shell
daytona env set
```

2. コンテナレジストリ用のサーバー環境変数キーを設定します。

```text
サーバー環境変数を設定
キー
>
```

:::tip
コンテナレジストリを構成するには、以下の環境変数を追加してください。

- **`<*>_CONTAINER_REGISTRY_SERVER`**

  コンテナレジストリの URL。

- **`<*>_CONTAINER_REGISTRY_USERNAME`**

  コンテナレジストリに認証するためのユーザー名。

- **`<*>_CONTAINER_REGISTRY_PASSWORD`**

  コンテナレジストリに認証するためのパスワード。

:::

<br />

3. コンテナレジストリ用のサーバー環境変数の値を設定します。

```text
値
>
```

4. さらに環境変数を追加するかどうかを選択します。

```text
さらに環境変数を追加しますか？

[はい]     [いいえ]
```

選択すると、Daytona が Workspace 用のコンテナレジストリ環境変数を設定します。

```text
サーバー環境変数が正常に設定されました
```

## コンテナレジストリの一覧表示

Daytona は、ワークスペースに設定されているコンテナレジストリを一覧表示するオプションを提供しています。

1. 次のコマンドを実行して、コンテナレジストリの一覧を表示します。

```shell
daytona env list
```

このコマンドを実行すると、Daytona は設定されているコンテナレジストリの環境変数の一覧を表示します。

```text
Key                         Value
────────────────────────────────────  
MyEnvVariable               ********
```

## コンテナレジストリを削除する

Daytona では、ワークスペースからコンテナレジストリを削除するオプションが用意されています。

1. 次のコマンドを実行して、コンテナレジストリを削除します。

```shell
daytona env delete
```

このコマンドを実行すると、削除したいコンテナレジストリの選択を求められます。

```text
削除するサーバー環境変数を選択してください
1 項目
===
MyEnvVariable
********
```

```text
サーバー環境変数が正常に削除されました
```
