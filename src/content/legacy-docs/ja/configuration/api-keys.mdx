Daytona API キーは、[Daytona API](/docs/tools/api) へのリクエストを認証するために使用されます。Daytona では、API リクエストを管理するための API キーの生成、生成されたすべての API キーの一覧表示、および API キーの取り消しを行うオプションが提供されています。

## API キーの生成

Daytona は API キーを生成するオプションを提供しています。

1. 以下のコマンドを実行して API キーを生成します：

```bash
daytona api-key generate
```

このコマンドを実行すると、Daytona は API キーの名前を入力するよう求めます。

```text
Name
>
```

API キーの生成が成功すると、Daytona は以下の確認メッセージを表示します：

```text
You have successfully generated a new API Key and Server URL, which you can find below:

DAYTONA_API_KEY=your-api-key
DAYTONA_SERVER_URL=your-server-url
```

Daytona は、以下のコマンドを実行することで、クライアントマシンから Daytona Server に即座に接続するオプションを提供しています：

```bash
daytona profile add -a <your-server-url> -k <your-api-key>
```

## API キーの一覧表示

Daytonaは生成されたすべてのAPIキーを一覧表示するオプションを提供します。

1. 生成されたすべてのAPIキーを一覧表示するには、以下のコマンドを実行してください：

```bash
daytona api-key list
```

このコマンドを実行すると、Daytonaは生成されたすべてのAPIキーの一覧を表示します。

```text
Name                                Type
──────────────────────────────────────────
default                             client
```

:::tip
Daytonaは既存のDaytona Server API URLを表示するオプションを提供します。

<br />

```bash
daytona server config
```

```text
API URL:  https://<server-id>.try-eu.daytona.app
```

:::

## API キーを取り消す

Daytona では API キーを取り消すオプションが提供されています。

1. 以下のコマンドを実行して API キーを取り消します：

```bash
daytona api-key revoke
```

このコマンドを実行すると、Daytona は取り消したい API キーの名前を入力するよう求めます。

```text
Select an API key to revoke
1 item
===
MyAPIKey
client
```

3. 選択した API キーで **`Enter`** を押して取り消します。

4. **`Yes`** または **`No`** を選択して取り消しを確認します。

```text
Revoke API Key 'test'?
Are you sure you want to revoke 'test'?

[Yes]     [No]
```

```text
API key revoked
```
