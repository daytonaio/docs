Daytona APIキーは、[Daytona API](/docs/ja/tools/api)へのリクエストを認証するために使用されます。Daytonaでは、APIリクエストを管理するためのAPIキーの生成、生成済みAPIキーの一覧表示、APIキーの無効化が可能です。

## APIキーを生成する

Daytona では、APIキーを生成するオプションが用意されています。

1. 次のコマンドを実行してAPIキーを生成します:

```bash
daytona api-key generate
```

このコマンドを実行すると、Daytona からAPIキーの名前の入力を求められます。

```text
Name
>
```

APIキーの生成に成功すると、Daytona は次の確認メッセージを表示します:

```text
You have successfully generated a new API Key and Server URL, which you can find below:

DAYTONA_API_KEY=your-api-key
DAYTONA_SERVER_URL=your-server-url
```

Daytona では、クライアントマシンから次のコマンドを実行することで、Daytona Server にすぐに接続するオプションも用意されています。

```bash
daytona profile add -a <your-server-url> -k <your-api-key>
```

## APIキーの一覧表示

Daytona では、生成されたすべてのAPIキーを一覧表示するオプションが用意されています。

1. 次のコマンドを実行して、生成されたすべてのAPIキーを一覧表示します。

```bash
daytona api-key list
```

このコマンドを実行すると、Daytonaは生成されたすべてのAPIキーの一覧を表示します。

```text
Name                                Type
──────────────────────────────────────────
default                             client
```

:::tip
Daytona では、既存のDaytona Server API URLを表示するオプションも用意されています。

<br />

```bash
daytona server config
```

```text
API URL:  https://<server-id>.try-eu.daytona.app
```

:::

## APIキーの取り消し

Daytona では、APIキーを取り消すオプションが用意されています。

1. APIキーを取り消すには、次のコマンドを実行します。

```bash
daytona api-key revoke
```

このコマンドを実行すると、Daytona から取り消したいAPIキーの名前を入力するように求められます。

```text
取り消すAPIキーを選択してください
1件
===
MyAPIKey
client
```

3. 取り消したいAPIキーを選択し、**`Enter`** キーを押します。

4. **`Yes`** または **`No`** を選択して、取り消しを確認します。

```text
APIキー 'test' を取り消しますか？
'test' を本当に取り消してもよろしいですか？

[Yes]     [No]
```

```text
APIキーが取り消されました
```
