ターゲット構成とは、さまざまな[プロバイダー](/docs/ja/configuration/providers)によって支援される開発環境がデプロイおよび管理される特定の宛先や環境を指します。プロバイダーは環境を作成するための方法や技術を定義し、ターゲット構成はこれらの環境が存在する正確な場所やプラットフォームを指定します。

ターゲット構成はローカルマシン、リモートサーバー、またはクラウドインスタンスである場合があり、選択したプロバイダーによって異なります。ターゲット構成を利用することで、さまざまなプラットフォームやアカウントにわたって環境をデプロイ・管理する柔軟性が得られます。これらはすべてDaytonaが提供する統一されたインターフェース上で行えます。

ターゲット構成を追加すると、Daytonaで新しい開発環境を作成する際に選択できるようになります。複数のターゲット構成を作成したり、既存のものを一覧表示したり、不要になったものを削除したりすることができます。

## ターゲット構成の作成

Daytona では、ワークスペースを管理する際に使用するターゲット構成を作成するオプションが用意されています。

1. 次のコマンドを実行してターゲット構成を作成します。

```shell
daytona target-config create
```

2. デプロイ先の環境に適したプロバイダーを選択します。

```text
プロバイダーを選択
7項目
===
Docker
v0.0.0
DigitalOcean
v0.0.0
AWS
v0.0.0
Azure
v.0.0.0
GCP
v0.0.0
Hetzner
v0.0.0
Fly
v0.0.0
```

3. ターゲット構成の名前を入力します。

```text
ターゲット構成名
>
```

4. プロンプトが表示されたら、適切な構成オプションを入力します。構成オプションは選択したプロバイダーによって異なります。次の例は、[リモート Docker ターゲット構成](#docker-remote)を追加する場合です。

```text
リモートホスト名
>

リモートパスワード
>

リモートポート
>

リモートユーザー
注: root 以外のユーザーが必要です
>

ソックパス
> /var/run/docker.sock

ワークスペースデータディレクトリ       
リモートホスト上でワークスペースデータが保存されるディレクトリ
> /tmp/daytona-data

リモート秘密鍵パス
daytona_config
known_hosts
カスタムパス
なし
```

5. **`Enter`** を押してターゲット構成の追加を確定します。

```text
ターゲット構成 <target_config> が正常に作成されました
```

## ターゲット構成の一覧

Daytona では、これまでに作成したターゲット構成をその詳細とともに一覧表示して管理するオプションが用意されています。

1. 現在設定されているターゲット構成を一覧表示するには、次のコマンドを実行します。

```shell
daytona target-config list
```

このコマンドを実行すると、Daytona はターゲット構成の一覧とその詳細を表示します。ターゲット構成名、接続されているプロバイダー、および設定されているランナーを確認できます。

```text
Target            Provider                Runner
────────────────────────────────────────────────
local             docker-provider         local
```

## ターゲット構成の削除

Daytona では、不要になったターゲット構成を管理・削除するためのオプションが用意されています。ターゲット構成を削除すると、そのターゲット構成上でワークスペースを管理したりデプロイしたりすることはできなくなります。

1. ターゲット構成を削除するには、次のコマンドを実行します。

```shell
daytona target-config delete
```

このコマンドを実行すると、Daytona はあなたのターゲット構成の一覧とその詳細を表示します。構成名、ワークスペースの一意の識別子、および接続されているリポジトリを確認できます。

2. 削除したい構成を選択し、**`Enter`** キーを押します。

```text
Choose a Target Config To Delete
1 item
===
MyTarget          
docker-provider
```

3. ターゲット構成の削除を確認するために **`Yes`** をクリックします。

```text
Delete all workspaces within MyTarget?
You might not be able to easily remove these workspaces later.

[Yes]     [No]
```

```text
Target Config MyTarget removed successfully
```

## Docker（ローカル）

Daytona は、Workspace のデプロイと管理に使用するローカル Docker ターゲット構成を作成するオプションを提供しています。ローカル Docker ターゲット構成を使用すると、Docker を使って自分のローカルマシン上で Workspace をデプロイできます。

:::note
ローカル Docker ターゲット構成を作成する前に、[Docker](https://docs.docker.com/engine/install/) がローカルマシンにインストールされ、起動していることを確認してください。
:::

<br />

1. ターゲット構成を作成するには、次のコマンドを実行します。

```shell
daytona target-config create
```

2. ローカル Docker プロバイダーを使用するために **`Docker`** を選択します。

```text
Choose a provider
1 item
===
Docker
v0.0.0
```

3. ターゲット構成の名前を入力します。

:::note
ローカル Docker ターゲット構成として認識されるためには、ターゲット構成の名前を `local` にする必要があります。
:::

<br />

```text
Target Config Name
>
```

4. 指示に従って適切な構成オプションを入力します。

```text
Sock Path
> /var/run/docker.sock
```

5. ターゲット構成の設定を確定するには **`Enter`** を押します。

```text
Target config <target_config_name> created successfully
```

## Docker（リモート）

Daytona は、リモート Docker ターゲット構成を作成してワークスペースのデプロイと管理に使用するオプションを提供します。リモート Docker ターゲット構成を利用することで、リモートマシン上で Docker を使ってワークスペースをデプロイできます。

:::

<br />

1. 次のコマンドを実行してターゲット構成を作成します：

```shell
daytona target-config add
```

2. Docker プロバイダーを使用するために **`Docker`** を選択します。

```text
>

Remote Password
>

Remote Port
>

Remote User
Note: non-root user required
>

Sock Path
> /var/run/docker.sock

Target Data Dir
The directory on the remote host where the target data will be stored
> /tmp/daytona-data

Remote Private Key Path
Showing files in: /home/user/.ssh
You can select a file, choose None or enter a Custom path

> config.save
> id_rsa
> id_rsa.pub
> id_rsa_gitlab
> id_rsa_gitlab.pub
> known_hosts
> known_hosts.old
> Custom path
> None
```

```text

GCP_CREDENTIAL_FILE 環境変数を設定している場合は空欄のままにしてください。
```
