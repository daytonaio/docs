Daytonaは、Apache License 2.0の下でライセンスされたオープンソースのDevelopment Environment Manager（DEM）です。
Daytonaを使うことで、標準のOCIコンテナに基づいた再現可能な開発環境であるWorkspaceを管理・デプロイできます。Dev Container標準もネイティブでサポートしています。
Daytonaのアーキテクチャは、将来的にDockerfile、Docker Compose、Nix、Devfileなど他の構成標準に基づいたWorkspaceの構築も可能にします。

プロジェクトのリモートGitリポジトリにある構成を活用することで、DaytonaはWorkspaceを構築し、お好みのプラットフォームにWorkspaceをプロビジョニングします。
プロビジョニングが完了したら、Visual Studio CodeなどDaytonaがサポートするIDEを使ってプロジェクトの開発やテストが行えます。

Daytonaの機能は、Linux、macOS、Windowsの各システム上で、x86-64およびAArch64アーキテクチャの両方に対応したコマンドラインツールを通じて利用できます。

## 機能

* __セキュリティ__

    Daytonaはクライアントマシンとリモートマシンの間に安全なVPN接続を作成します。
    リモートマシン上のすべてのポートに、手動でポートフォワーディングを行うことなく安全にアクセスできます。

* __Visual Studio CodeおよびJetBrainsのサポート__

    DaytonaはVisual Studio CodeとJetBrains系のIDEの両方をサポートしており、すべてがローカルのように感じながらプロジェクトの開発を簡単に行えます。

* __GitHub、GitLab、Bitbucket、Giteaとの連携__

    Daytonaは、お好みのSCMプラットフォームからリポジトリを取得してDEを作成できます。Git操作はワークスペース内で実行でき、コンテキストを切り替えることなく作業をプッシュできます。

* __マルチプロジェクトワークスペースのサポート__

    Daytonaは複数のプロジェクトを含むワークスペースを作成できます。
    マイクロサービスや複数のリポジトリに分割された大規模プロジェクトも、1つのワークスペースで作業できます。

* __リバースプロキシのサポート__

    Daytonaはリバースプロキシを統合しており、パブリックまたは制限されたネットワーク上のワークスペースにアクセスできます。

* __拡張可能なコア__

    DaytonaはGoで開発されたプラグインをサポートしています。
    サードパーティプロバイダーやコア機能の拡張をDaytonaに追加することができます。
