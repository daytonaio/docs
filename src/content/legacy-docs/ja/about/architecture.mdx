## Open Source

Daytona Open Sourceには、Daytonaとプログラム的にやり取りするための中央インターフェースとして機能する[Daytona API](/docs/tools/api)が含まれています。Open Sourceバージョンは、[インターフェース](#interfaces)、[Git Providers](#git-providers)、[Container Registries](#container-registries)、[Providers](#providers)、[Targets](#targets)を含む、開発環境を管理するための包括的な機能とツールセットを提供します。

### Interfaces

Daytona Open Sourceは、プラットフォームとやり取りし、開発環境を管理するための様々な[インターフェース](/docs/usage/ide)を提供しています。インターフェースはDaytonaとやり取りする異なる方法を提供し、開発者が自分の好みやワークフローに基づいて最も適切な方法を選択できるようにします。

これらには、直接制御のための[コマンドラインインターフェース（CLI）](/docs/tools/cli)、[Visual Studio Code（VSCode）](/docs/usage/ide#vs-code)、[Visual Studio Code（VSCode）Browser](/docs/usage/ide#vs-code-browser)、[Visual Studio Code Insiders](/docs/usage/ide#vs-code-insiders)、[Visual Studio Codium](/docs/usage/ide#vscodium)、[Jupyter](/docs/usage/ide#jupyter)、[Zed](/docs/usage/ide#zed)、[Cursor](/docs/usage/ide#cursor)、[Headless](/docs/usage/ide#headless)、[Windsurf](/docs/usage/ide#windsurf)などの[IDE](/docs/usage/ide)、統合開発のためのJetBrains IDE、および[Terminal SSH](/docs/usage/ide#terminal-ssh)が含まれます。

さらに、開発者がカスタム統合を作成し、プラットフォーム機能を拡張できるように[Daytona SDK](/docs/sdk)が提供されています。

### Git Providers

Daytona Open SourceはGit Providersを統合して、バージョン管理操作を管理し、Gitリポジトリとやり取りします。Git Providersは効率的なソースコード管理とWorkspacesとの同期を可能にします。

サポートされているGit Providersには、[GitHub](/docs/configuration/git-providers#github)、[GitLab](/docs/configuration/git-providers#gitlab)、[Bitbucket](/docs/configuration/git-providers#bitbucket)、[GitHub Enterprise Server](/docs/configuration/git-providers#github-enterprise-server)、[GitLab Self-Managed](/docs/configuration/git-providers#gitlab-self-managed)、[Bitbucket Server](/docs/configuration/git-providers#bitbucket-server)、[Codeberg](/docs/configuration/git-providers#codeberg)、[Gitea](/docs/configuration/git-providers#gitea)、[Gitness](/docs/configuration/git-providers#gitness)、[Azure DevOps](/docs/configuration/git-providers#azure-devops)、[AWS CodeCommit](/docs/configuration/git-providers#aws-codecommit)、[Gogs](/docs/configuration/git-providers#gogs)、[Gitee](/docs/configuration/git-providers#gitee)があります。

Git Providers統合は、協調開発ワークフローを促進し、リポジトリと開発環境間の接続を合理化します。

### Providers

Daytona Open Sourceは[Providers](/docs/configuration/providers)を使用して、開発環境を作成・管理するための様々な技術との統合を可能にします。Providersは基盤技術の複雑さを抽象化し、Daytonaが環境をデプロイ・実行するために活用する基盤エンジンとして機能します。

サポートされているProvidersには、[Docker](/docs/configuration/providers#docker)、[DigitalOcean](/docs/configuration/providers#digitalocean)、[AWS](/docs/configuration/providers#aws)、[Azure](/docs/configuration/providers#azure)、[GCP](/docs/configuration/providers#gcp)、[Hetzner](/docs/configuration/providers#hetzner)、[Fly](/docs/configuration/providers#fly)があります。

### Targets

[Targets](/docs/usage/targets)は、Workspaceが実行される分離された処理単位を表します。クラウドベースやオンプレミス環境を含む、さまざまなインフラストラクチャ上で動作できます。Targetsは必要な計算リソースを提供し、定義されたインフラストラクチャ内でセキュリティ要件を満たします。

## Enterprise

DaytonaのEnterprise版は、オープンソース版の機能を拡張し、エンタープライズグレードの開発環境に特化した追加機能とサービスを提供します。これらの機能強化には、高度なセキュリティ、認証、認可メカニズム、さらにワークスペースのパフォーマンスと利用率を最適化するための可観測性とリソース管理ツールが含まれます。

### Dashboard

Enterprise版では、プラットフォームの管理と監視のための一元化されたインターフェースを提供するDashboardが導入され、開発環境の可視性と制御が向上します。

### Authentication

認証機能は、シングルサインオン（SSO）およびSecurity Assertion Markup Language（SAML）やSystem for Cross-domain Identity Management（SCIM）などの標準をサポートし、プラットフォームへの安全なアクセスを確保します。これにより、エンタープライズアイデンティティシステムとのシームレスな統合が可能になり、ユーザー管理が簡素化されます。

### Authorization

認可機能により、ユーザー、チーム、組織、プロジェクト全体でアクセスと権限の詳細な制御が可能になります。これにより、ロールベースのアクセス管理を提供することで、安全なコラボレーションと組織ポリシーの遵守が確保されます。

### Observability

可観測性ツールは、Workspacesと開発活動に対する強化された監視、ログ記録、トレース機能を提供します。これにより、チームはシステムパフォーマンスを追跡し、問題を効果的にデバッグし、環境に関する運用インサイトを維持できます。

### Resource Management

Resource Managementは、エンタープライズ機能により拡張され、大規模で組織全体の要件を満たすためにワークスペースクラスとクォータ管理が強化されます。

### Identity Providers

Enterprise版は、認証とアイデンティティ管理を強化するためにIdentity Providers（IDP）と統合されます。サポートされるIDPには、EntraID、Cognito、Okta、OpenID、Google、その他が含まれます。これらの統合により、企業は既存のアイデンティティシステムを使用してユーザー認証を管理でき、プラットフォームへの安全でコンプライアンスに準拠したアクセスが可能になります。

### Security

Daytona Enterpriseは、高度なセキュリティツールとサービスとの統合により、セキュリティを強化します。これらには、JFrog、Snyk、Sonatype、その他が含まれ、依存関係とコンテナ化された環境の脆弱性が特定され、軽減されることを確保します。これらのセキュリティ統合は、自動化されたスキャン、レポート、修復を提供し、開発ワークフローとアプリケーションの整合性を強化します。
