Daytona へようこそ！このガイドでは、Daytona の始め方についてご案内します。Daytona は、開発環境の作成・管理・デプロイを簡単にするために設計された、オープンソースの Development Environment Manager（DEM）です。

## Daytonaの理解

Daytonaは、Apache License 2.0の下でライセンスされたオープンソースのDevelopment Environment Manager（DEM）です。Daytonaを使うことで、標準のOCIコンテナに基づいた再現可能な開発環境である[Workspaces](/docs/ja/usage/workspaces)を管理・デプロイできます。また、Dev Container標準もネイティブにサポートしています。

## はじめに

Daytona をインストールして使用する前に、お使いのオペレーティングシステムが必要な要件、依存関係、およびツールを満たしていることを確認してください。

### システム要件

Daytona は複数のオペレーティングシステムとアーキテクチャをサポートしています。お使いのシステムが以下のいずれかの仕様に合致していることを確認してください。

| **オペレーティングシステム**                         | **アーキテクチャ**                                   |
|-----------------------------------------------| --------------------------------------------------|
| [**Linux**](/docs/ja/installation/installation)      | **`x86_64`** または **`ARM64`**                               |
| [**macOS**](/docs/ja/installation/installation)      | **`x86_64 (Intel-based)`** または **`ARM64 (Apple Silicon)`** |
| [**Windows**](/docs/ja/installation/installation)  | **`x86_64`** または **`ARM64`**                               |

### 依存関係

Daytona では、開発環境の運用と管理をサポートするために、特定の依存関係がシステムにインストールされている必要があります。これらの依存関係がシステムにインストールされ、適切に設定されていることを確認してください。

| **依存関係**            | **説明**                                                                                 |
|---------------------------|-------------------------------------------------------------------------------------------------|
| <u>[**Docker**](https://docs.docker.com/engine/install/)</u> | Daytona は Docker を使用して、分離された開発環境を作成および管理します。                     |
| 管理者権限  | Daytona をシステムにインストールおよび設定するために必要な権限があることを確認してください。 |

### ツール

Daytona では、開発環境のセットアップと管理を最適化するために、特定のツールがシステムにインストールされている必要があります。これらのツールがシステムにインストールされ、適切に設定されていることを確認してください。

| **ツール**                          | **説明**                                                                                                                                 |
|-----------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| テキストエディタ / IDE                 | 設定ファイルの編集や Daytona Workspaces 内でのコード作業に推奨されます。                                                    |
| SSH クライアント                        | リモートサーバーへアクセスする際に必要です。特に、リモートの Docker 環境で Workspaces をデプロイする場合や、リモートの Git リポジトリへアクセスする場合に必要です。 |

## CLIの使用方法

Daytona コマンドラインインターフェース（CLI）は、Daytona プラットフォームとやり取りするための主な方法です。CLI は、ワークスペース、プロバイダー、その他のシステム設定を管理できる包括的なコマンドセットを提供します。

Daytona CLI にアクセスするには、ターミナルを開き、`daytona` コマンドの後に目的のサブコマンドを入力します。CLI には幅広いコマンドが用意されており、それぞれ特定の目的（ワークスペースの作成、プロバイダーの管理、ターゲットの設定、システム設定の構成など）に役立ちます。

利用可能なコマンドとその説明の完全な一覧については、[CLIリファレンス](/docs/ja/tools/cli) を参照してください。このリファレンスガイドでは、各コマンドの詳細情報、使用例、コマンドオプションなどが記載されており、Daytona CLI の機能を最大限に活用するのに役立ちます。

`daytona --help` コマンドを使用すると、すべてのコマンドの一覧が表示され、利用可能なオプションを簡単に確認できます。

## Gitプロバイダーの追加

Daytonaは、お好みの[Gitプロバイダー](/docs/ja/configuration/git-providers)を統合するオプションを提供しており、開発ワークフローを強化します。この統合により、Daytona内でリポジトリへ直接アクセスし管理できるようになり、既存プロジェクトからのワークスペース作成が簡単になります。

Daytonaは、[GitHub](/docs/ja/configuration/git-providers#github)、[GitLab](/docs/ja/configuration/git-providers#gitlab)、[Bitbucket](/docs/ja/configuration/git-providers#bitbucket)、[GitHub Enterprise Server](/docs/ja/configuration/git-providers#github-enterprise-server)、[GitLab Self-Managed](/docs/ja/configuration/git-providers#gitlab-self-managed)、[Bitbucket Server](/docs/ja/configuration/git-providers#bitbucket-server)、[Codeberg](/docs/ja/configuration/git-providers#codeberg)、[Gitea](/docs/ja/configuration/git-providers#gitea)、[Gitness](/docs/ja/configuration/git-providers#gitness)、[Azure DevOps](/docs/ja/configuration/git-providers#azure-devops)、[AWS CodeCommit](/docs/ja/configuration/git-providers#aws-codecommit)、[Gogs](/docs/ja/configuration/git-providers#gogs)、[Gitee](/docs/ja/configuration/git-providers#gitee)など、さまざまなGitプロバイダーをサポートしています。

1. Gitプロバイダーを追加するには、次のコマンドを実行します:

```shell
daytona git-providers add
```

2. 追加したいGitプロバイダーをリストから選択します。

3. Daytonaが認証し、リポジトリへの安全なアクセスを有効にするため、Gitプロバイダーの認証情報を入力します。

認証が成功すると、DaytonaはGitプロバイダーを登録し、Daytonaから直接リポジトリとやり取りできるようになります。

```text
Git provider has been registered
```

複数のプラットフォームやアカウントにまたがるリポジトリを管理するために、追加の[Gitプロバイダー](/docs/ja/configuration/git-providers)をインストールすることもできます。

`daytona git-providers --help`コマンドを使用すると、Gitプロバイダーの設定管理やリポジトリとのやり取りに利用できるオプションを確認できます。

## プロバイダーのインストール

[プロバイダー](/docs/ja/configuration/providers)は、Daytonaがさまざまな技術と統合し、開発環境を作成・管理するためのプラグインです。プロバイダーは基盤となる技術の複雑さを抽象化し、Daytonaがコンテナ化、オーケストレーション、またはクラウドベースの仮想マシンを通じて環境をデプロイおよび実行するための基礎的なエンジンとして機能します。

Daytonaは、[Docker](/docs/ja/configuration/providers#docker)、[DigitalOcean](/docs/ja/configuration/providers#digitalocean)、[AWS](/docs/ja/configuration/providers#aws)、[Azure](/docs/ja/configuration/providers#azure)、[GCP](/docs/ja/configuration/providers#gcp)、[Hetzner](/docs/ja/configuration/providers#hetzner)、[Fly](/docs/ja/configuration/providers#fly) など、さまざまなプロバイダーをサポートしています。

1. プロバイダーをインストールするには、次のコマンドを実行します:

  ```shell
  daytona provider install
  ```

2. インストールしたいプロバイダーをリストから選択します。

プロバイダーを選択すると、Daytonaがそのプロバイダーをインストールおよび設定し、開発環境の管理とデプロイに利用できるようにします。

```text
Provider <provider> has been successfully installed
```

追加の[プロバイダー](/docs/ja/configuration/providers)をインストールすることで、Daytonaの機能を拡張し、幅広いコンテナ管理プラットフォームやクラウドホスティングサービスをサポートできます。

## ターゲット構成の作成

[ターゲット構成](/docs/ja/configuration/target-config)とは、さまざまな[プロバイダー](/docs/ja/configuration/providers)によって支援される開発環境がデプロイおよび管理される特定の宛先や環境を指します。プロバイダーは環境を作成するための方法や技術を定義し、ターゲット構成はこれらの環境が存在する正確な場所やプラットフォームを指定します。

ターゲット構成はローカルマシン、リモートサーバー、またはクラウドインスタンスである場合があり、選択したプロバイダーによって異なります。ターゲット構成を利用することで、さまざまなプラットフォームやアカウントにわたって環境をデプロイおよび管理する柔軟性が得られます。これらはすべてDaytonaが提供する統一されたインターフェース上で行えます。

サポートされているターゲット構成には、[Docker（ローカル）](/docs/ja/configuration/target-config#docker-local)、[Docker（リモート）](/docs/ja/configuration/target-config#docker-remote)、[DigitalOcean](/docs/ja/configuration/target-config#digitalocean)、[AWS](/docs/ja/configuration/target-config#aws)、[Azure](/docs/ja/configuration/target-config#azure)、[GCP](/docs/ja/configuration/target-config#gcp)、[Hetzner](/docs/ja/configuration/target-config#hetzner)、および[Fly](/docs/ja/configuration/target-config#fly)が含まれます。

1. 次のコマンドを実行してターゲット構成を作成します:

```shell
daytona target-config create
```

2. デプロイしたい環境に適したプロバイダーを選択します。

3. ターゲット構成の名前を入力します。

4. 指示に従って適切な構成オプションを入力します。構成オプションは選択したプロバイダーによって異なります。

5. **`Enter`** を押してターゲット構成の追加を確定します。

```text
Target Config <target_config> created successfully
```

`daytona target-config --help` コマンドを使用すると、ターゲット構成を設定するために利用可能な構成オプションを確認できます。

## コンテナレジストリの設定

[コンテナレジストリ](/docs/ja/configuration/container-registries) は、指定されたレジストリサーバーからコンテナイメージを取得するために使用される認証情報を保存します。コンテナレジストリの認証情報を追加することで、プライベートイメージやプライベートレジストリにホストされているイメージから Workspace を作成したいユーザーにとって便利です。

1. 次のコマンドを実行して、コンテナレジストリの環境変数を設定します。

```shell
daytona env set
```

2. コンテナレジストリのサーバー環境変数キーを設定します。

3. コンテナレジストリのサーバー環境変数値を設定します。

4. 別の環境変数を追加するかどうかを選択します。

選択すると、Daytona は Workspace のためにコンテナレジストリの環境変数を設定します。

```text
サーバー環境変数が正常に設定されました
```

利用可能な環境変数設定オプションを確認するには、`daytona env --help` コマンドを使用してください。

## ワークスペーステンプレートの作成

[ワークスペーステンプレート](/docs/ja/configuration/workspace-templates) は、開発環境の管理を簡素化するために設計されており、あらかじめ設定を定義しておくことで、新しいワークスペースに簡単に適用できるようになります。

ワークスペーステンプレートには、ワークスペース内でワークスペースを定義・管理するために必要なすべてのプロパティが含まれています。リポジトリのURL、ビルド構成、環境変数などの情報を保存します。これにより、以前に設定したテンプレートを簡単に再利用でき、複数のワークスペースで一貫性のある再現可能なセットアップを実現できます。

1. 次のコマンドを実行してワークスペーステンプレートを追加します:

```shell
daytona workspace-template add
```

2. ワークスペーステンプレートを作成するために用意された2つのオプションのいずれかを選択します:

- **カスタムリポジトリURLを入力**

   リポジトリがGitプロバイダーアカウントに表示されていない場合や外部ソースの場合に、カスタムリポジトリURLを手動で入力するオプションです。

- **サンプルから作成**

   まだGitリポジトリが用意できていない場合は、Daytonaが提供するサンプルプロジェクトからワークスペーステンプレートを作成できます。

3. 設定したいワークスペースのリポジトリURLを入力するか、サンプルプロジェクトからワークスペーステンプレートを作成するオプションを選択します。

4. ワークスペーステンプレートの名前を入力します。

5. ワークスペースのビルド構成を選択します。

6. ワークスペースの環境変数を **`KEY=VALUE`** 形式で入力します。

完了すると、Daytonaがワークスペーステンプレートの概要を出力し、保存前にプロパティを確認できるようになります。

```text
SUMMARY - Workspace Template

Workspace - MyWorkspace
Repository  https://github.com/daytonaio/docs
Branch      main 
Build       Automatic
Image       daytonaio/workspace-project:latest
User        daytona
```

7. **`Create`** をクリックしてワークスペーステンプレートを保存するか、**`Configure`** をクリックしてワークスペーステンプレートを変更します。

```text
Workspace template <workspace_template> added successfully
```

`daytona workspace-template --help` コマンドを使用すると、ワークスペーステンプレート作成時に利用可能な設定オプションを確認できます。

## ターゲットの設定

[Target](/docs/ja/usage/targets) とは、Workspace が実行される独立した処理ユニットです。これは、クラウドベースやオンプレミス環境など、さまざまなインフラストラクチャ上で動作させることができます。ターゲットは、必要なコンピュートリソースを提供し、定義されたインフラストラクチャ内でセキュリティ要件を満たします。

1. 次のコマンドを実行してターゲットを作成します:

```shell
daytona target create
```

2. ターゲットで使用する [Target Configuration](/docs/ja/configuration/target-config) を選択します。

3. デプロイ先の環境に合わせて [Provider](/docs/ja/configuration/providers) を選択します。

4. ターゲットの名前を入力します。

5. プロンプトに従って適切な構成オプションを入力します。構成オプションは選択した Provider によって異なります。

```text
✓ リクエストが送信されました
ターゲット <target_name> を作成中
```

異なる開発環境ごとに追加の [Targets](/docs/ja/usage/targets) を設定することができ、必要に応じてそれらを管理・切り替えることが可能です。

`daytona target --help` コマンドを使用すると、ターゲット設定のために利用可能な構成オプションを確認できます。

## Runner の作成

Runner は、どこにでもデプロイできるスタンドアロンのサービスです。ユーザーは、Runner 登録時に生成される一連の手順とコマンドを実行することで、ターゲットシステムに Runner をデプロイできます。

1. 新しい Runner を作成するには、次のコマンドを実行します。

```shell
daytona server runner create
```

このコマンドを実行すると、Daytona はターゲットシステムに Runner をデプロイするために必要な server API URL と API キーを含むコードスニペットを生成します。

2. 生成されたコードスニペットをターゲットシステムのターミナルに貼り付けて実行し、Runner を構成します。

1. デフォルトのIDEを設定するには、次のコマンドを実行します。

```shell
daytona ide
```

2. 表示されたリストから希望するIDEを選択します。選択すると、DaytonaがそれをデフォルトIDEとして設定し、確認メッセージが表示されます。

```text

```

`daytona code` コマンドを使うと、すでにWorkspaceに接続されたデフォルトIDEを自動的に開くことができます。

追加の[IDE](/docs/ja/configuration/git-providers)を選択し、開発環境のデフォルトとして設定することも可能です。

`daytona ide --help` コマンドを使うと、IDEの利用可能な設定オプションを確認できます。

## ワークスペースの作成

[ワークスペースを作成する](/docs/ja/usage/workspaces#create-a-workspace)ことで、Daytona を使って開発環境を管理できます。ワークスペースは、コードの管理、プロジェクトのビルド、チームとのやり取りができる、分離された開発環境を表します。

ワークスペースを作成するには、`daytona create` コマンドを使用し、プロンプトに従ってワークスペースの設定を行います。作成後は、ワークスペースにアクセスしてコードベースとやり取りできます。

1. 次のコマンドを実行して、ガイド付きのワークスペース作成プロセスを開始します。

```shell
daytona create
```

2. 提供されている3つのオプションから1つを選択します。

- **リポジトリ一覧から選択**
- **カスタムリポジトリURLを入力**
- **サンプルから作成**

3. リストからGitリポジトリを選択するか、ワークスペースにリンクしたいGitリポジトリのURLを入力するか、サンプルプロジェクトからワークスペースを作成することを選択します。

4. ワークスペースの名前を入力します。

5. ワークスペースをデプロイする[ターゲット](/docs/ja/usage/targets)を選択します。ターゲットを選択すると、[ターゲット設定](/docs/ja/configuration/target-config)の入力が求められます。設定内容は選択したターゲットによって異なります。

ターゲットの選択と設定が完了すると、Daytona は指定したGitリポジトリをクローンし、その内容に基づいて新しいワークスペースをセットアップします。ターミナルには、ワークスペース作成の進行状況が表示されます。

```text
<Target>    | ✓ リクエストが送信されました
<Workspace> | ワークスペース <workspace> (<workspace_id>) を作成中
<Workspace> | イメージを取得中...
<Workspace> | daytonaio/workspace-project から取得中
<Workspace> | ダイジェスト: sha256:<sha256>
<Workspace> | ステータス: daytonaio/workspace-project:latest のイメージは最新です
<Workspace> | ✓ イメージの取得に成功しました
<Workspace> | UIDs と GIDs は同じです (1000:1000)。
<Workspace> | '/workdir/<workspace>' にクローン中...
<Workspace> | UIDs と GIDs は同じです (1000:1000)。
<Workspace> | ✓ ワークスペース <workspace> が作成されました
```

`daytona create --help` コマンドを使用すると、ワークスペース作成時に利用可能な設定オプションを確認できます。

## ワークスペースと対話する

[Agent Toolbox](/docs/ja/usage/agent-toolbox) を使うことで、ワークスペース内でさまざまな操作をプログラム的に実行できる Workspace Toolbox API を利用して、ワークスペースと対話できます。

Workspace Toolbox API はファイルシステム、Git、プロセスの操作を提供し、ワークスペース内でファイルの管理、Git コマンドの実行、プロセスの実行が可能です。
