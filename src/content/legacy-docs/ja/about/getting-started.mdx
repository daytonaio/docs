Daytonaへようこそ！このガイドでは、開発環境の作成、管理、デプロイメントを簡素化するために設計されたオープンソースのDevelopment Environment Manager（DEM）であるDaytonaの使用開始方法をご案内します。

## Daytonaを理解する

DaytonaはApache License 2.0の下でライセンスされたオープンソースの開発環境マネージャー（DEM）です。Daytonaを使用すると、[Workspaces](/docs/ja/usage/workspaces)を管理およびデプロイできます。これらは標準のOCIコンテナに基づく再現可能な開発環境で、Dev Container標準の組み込みサポートを備えています。

## 始める前に

Daytonaをインストールして使用する前に、お使いのオペレーティングシステムが必要な要件、依存関係、およびツールを満たしていることを確認してください。

### システム要件

Daytonaは複数のオペレーティングシステムとアーキテクチャをサポートしています。お使いのシステムが以下の仕様のいずれかと一致することを確認してください：

| **オペレーティングシステム**                         | **アーキテクチャ**                                   |
|-----------------------------------------------| --------------------------------------------------|
| [**Linux**](/docs/ja/installation/installation)      | **`x86_64`** または **`ARM64`**                               |
| [**macOS**](/docs/ja/installation/installation)      | **`x86_64 (Intel-based)`** または **`ARM64 (Apple Silicon)`** |
| [**Windows**](/docs/ja/installation/installation)  | **`x86_64`** または **`ARM64`**                               |

### 依存関係

Daytonaは開発環境の操作と管理をサポートするために、特定の依存関係がシステムにインストールされている必要があります。これらの依存関係がシステムにインストールされ、適切に設定されていることを確認してください：

| **依存関係**            | **説明**                                                                                 |
|---------------------------|-------------------------------------------------------------------------------------------------|
| <u>[**Docker**](https://docs.docker.com/engine/install/)</u> | Daytonaは分離された開発環境を作成・管理するためにDockerを使用します。                     |
| 管理者権限  | システムにDaytonaをインストールして設定するために必要な権限があることを確認してください。 |

### ツール

Daytonaは開発環境のセットアップと管理を最適化するために、特定のツールがシステムにインストールされている必要があります。これらのツールがシステムにインストールされ、適切に設定されていることを確認してください：

| **ツール**                          | **説明**                                                                                                                                 |
|-----------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| テキストエディタ / IDE                 | 設定ファイルの編集やDaytona Workspace内でのコード作業に推奨されます。                                                    |
| SSHクライアント                        | リモートサーバーへのアクセスに必要です。特にリモートDockerインストール上でWorkspaceをデプロイする場合や、リモートGitリポジトリにアクセスする場合に必要です。 |

## CLIの使用

Daytonaコマンドラインインターフェース（CLI）は、Daytonaプラットフォームとやり取りする主要な方法です。CLIは、Workspace、Provider、その他のシステム設定を管理できる包括的なコマンドセットを提供します。

Daytona CLIにアクセスするには、ターミナルを開き、`daytona`コマンドに続けて目的のサブコマンドを使用します。CLIは幅広いコマンドを提供し、それぞれがWorkspaceの作成、Providerの管理、Targetの設定、システム設定の構成など、特定の目的を果たします。

利用可能なコマンドとその説明の完全なリストについては、[CLIリファレンス](/docs/ja/tools/cli)を参照してください。このリファレンスガイドでは、使用例やコマンドオプションを含む各コマンドの詳細情報を提供し、Daytona CLIの潜在能力を最大限に活用できるよう支援します。

`daytona --help`コマンドを使用してすべてのコマンドを一覧表示し、利用可能なオプションを探索する便利な方法を提供します。

## Git Providerの追加

Daytonaは、お好みの[Git Provider](/docs/ja/configuration/git-providers)を統合するオプションを提供し、開発ワークフローを向上させます。この統合により、Daytona内で直接リポジトリにアクセスし管理することができ、既存のプロジェクトからのワークスペース作成を簡素化します。

Daytonaは、[GitHub](/docs/ja/configuration/git-providers#github)、[GitLab](/docs/ja/configuration/git-providers#gitlab)、[Bitbucket](/docs/ja/configuration/git-providers#bitbucket)、[GitHub Enterprise Server](/docs/ja/configuration/git-providers#github-enterprise-server)、[GitLab Self-Managed](/docs/ja/configuration/git-providers#gitlab-self-managed)、[Bitbucket Server](/docs/ja/configuration/git-providers#bitbucket-server)、[Codeberg](/docs/ja/configuration/git-providers#codeberg)、[Gitea](/docs/ja/configuration/git-providers#gitea)、[Gitness](/docs/ja/configuration/git-providers#gitness)、[Azure DevOps](/docs/ja/configuration/git-providers#azure-devops)、[AWS CodeCommit](/docs/ja/configuration/git-providers#aws-codecommit)、[Gogs](/docs/ja/configuration/git-providers#gogs)、[Gitee](/docs/ja/configuration/git-providers#gitee)を含む様々なGit Providerをサポートしています。

1. 以下のコマンドを実行してGit Providerを追加します：

```shell
daytona git-providers add
```

2. リストから追加したいGit Providerを選択します。

3. Daytonaが認証し、リポジトリへの安全なアクセスを有効にするために、Git Providerの認証情報を入力します。

認証が成功すると、DaytonaはGit Providerを登録し、Daytonaから直接リポジトリとやり取りできるようになります。

```text
Git provider has been registered
```

複数のプラットフォームやアカウントにわたって複数のリポジトリを管理するために、追加の[Git Providers](/docs/ja/configuration/git-providers)をインストールできます。

`daytona git-providers --help`コマンドを使用して、Git Provider設定の管理とリポジトリとのやり取りに利用可能なオプションを表示します。

## Providerのインストール

[Providers](/docs/ja/configuration/providers)は、Daytonaが様々な技術と統合して開発環境を作成・管理するためのプラグインです。Providerは基盤技術の複雑さを抽象化し、Daytonaがコンテナ化、オーケストレーション、またはクラウドベースの仮想マシンを通じて環境をデプロイ・実行するために活用する基盤エンジンとして機能します。

Daytonaは[Docker](/docs/ja/configuration/providers#docker)、[DigitalOcean](/docs/ja/configuration/providers#digitalocean)、[AWS](/docs/ja/configuration/providers#aws)、[Azure](/docs/ja/configuration/providers#azure)、[GCP](/docs/ja/configuration/providers#gcp)、[Hetzner](/docs/ja/configuration/providers#hetzner)、[Fly](/docs/ja/configuration/providers#fly)を含む様々なProviderをサポートしています。

1. 以下のコマンドを実行してProviderをインストールします：

  ```shell
  daytona provider install
  ```

2. リストからインストールしたいProviderを選択します。

Providerを選択すると、Daytonaは選択されたProviderをインストール・設定し、開発環境の管理とデプロイに利用できるようにします。

```text
Provider <provider> has been successfully installed
```

追加の[Providers](/docs/ja/configuration/providers)をインストールして、Daytonaの機能を拡張し、幅広いコンテナ管理プラットフォームやクラウドホスティングサービスをサポートできます。

## Target Configurationの作成

[Target Configuration](/docs/ja/configuration/target-config)とは、様々な[Providers](/docs/ja/configuration/providers)によって促進される開発セットアップがデプロイされ管理される特定の宛先または環境を指します。Providersは環境を作成するために使用される方法と技術を定義し、Target Configurationsはこれらの環境が存在する正確な場所またはプラットフォームを指定します。

Target Configurationは、ローカルマシン、リモートサーバー、またはクラウドインスタンスであり、選択されたProviderに基づいて異なる場合があります。Target Configurationsは、Daytonaが提供する統一されたインターフェース内で、異なるプラットフォームやアカウント間で環境をデプロイし管理する柔軟性を提供します。

サポートされているTarget Configurationsには、[Docker (Local)](/docs/ja/configuration/target-config#docker-local)、[Docker (Remote)](/docs/ja/configuration/target-config#docker-remote)、[DigitalOcean](/docs/ja/configuration/target-config#digitalocean)、[AWS](/docs/ja/configuration/target-config#aws)、[Azure](/docs/ja/configuration/target-config#azure)、[GCP](/docs/ja/configuration/target-config#gcp)、[Hetzner](/docs/ja/configuration/target-config#hetzner)、および[Fly](/docs/ja/configuration/target-config#fly)があります。

1. 以下のコマンドを実行してTarget Configurationを作成します：

```shell
daytona target-config create
```

2. デプロイしたい環境に適したProviderを選択します。

3. Target Configurationの名前を入力します。

4. プロンプトが表示されたら、適切な設定オプションを入力します。設定オプションは選択されたProviderによって異なります。

5. **`Enter`**をクリックしてTarget Configurationの追加を確認します。

```text
Target Config <target_config> created successfully
```

Target Configurationを設定するための利用可能な設定オプションを表示するには、`daytona target-config --help`コマンドを使用してください。

## コンテナレジストリの設定

[コンテナレジストリ](/docs/ja/configuration/container-registries)は、指定されたレジストリサーバーからコンテナイメージをプルするために使用される認証情報を保存します。コンテナレジストリの認証情報を追加することは、プライベートイメージやプライベートレジストリでホストされているイメージからWorkspaceを作成したいユーザーにとって有用です。

1. 以下のコマンドを実行して、コンテナレジストリの環境変数を設定します：

```shell
daytona env set
```

2. コンテナレジストリのサーバー環境変数キーを設定します。

3. コンテナレジストリのサーバー環境変数値を設定します。

4. 別の環境変数を追加するかどうかを選択します。

選択すると、DaytonaはあなたのWorkspace用のコンテナレジストリ環境変数を設定します。

```text
Server environment variables have been set successfully
```

`daytona env --help`コマンドを使用して、環境変数設定の利用可能な設定オプションを表示します。

## Workspace Templateの作成

[Workspace Template](/docs/ja/configuration/workspace-templates)は、開発環境の管理を簡素化するために設計されており、新しいWorkspaceに簡単に適用できる事前定義された設定を可能にします。

Workspace Templateには、Workspace内でWorkspaceを定義および管理するために必要なすべてのプロパティが含まれています。リポジトリURL、ビルド設定、環境変数などの情報を保存します。これにより、以前に設定されたテンプレートの簡単な再利用が可能になり、複数のWorkspace間で一貫性のある再現可能なセットアップが保証されます。

1. 次のコマンドを実行してWorkspace Templateを追加します：

```shell
daytona workspace-template add
```

2. Workspace Templateを作成するために提供されている2つのオプションのうち1つを選択します：

- **カスタムリポジトリURLを入力**

   リポジトリがGit Providerアカウントにリストされていない場合や外部ソースからのものである場合に、カスタムリポジトリURLを手動で入力するオプション。

- **サンプルから作成**

   Gitリポジトリの準備ができていない場合は、Daytonaが提供するサンプルプロジェクトからWorkspace Templateを作成できます。

3. 設定したいWorkspaceのリポジトリURLを入力するか、サンプルプロジェクトからWorkspace Templateを作成することを選択します。

4. Workspace Templateの名前を入力します。

5. Workspaceのビルド設定を選択します。

6. **`KEY=VALUE`**形式でWorkspaceの環境変数を入力します。

完了すると、DaytonaはWorkspace Templateの概要を出力し、保存前にプロパティを確認できるようになります。

```text
SUMMARY - Workspace Template

Workspace - MyWorkspace
Repository  https://github.com/daytonaio/docs
Branch      main 
Build       Automatic
Image       daytonaio/workspace-project:latest
User        daytona
```

7. **`Create`**をクリックしてWorkspace Templateを保存するか、**`Configure`**をクリックしてWorkspace Templateに変更を加えます。

```text
Workspace template <workspace_template> added successfully
```

`daytona workspace-template --help`コマンドを使用して、Workspace Templateを作成するための利用可能な設定オプションを表示します。

## Targetの設定

[Target](/docs/ja/usage/targets)は、Workspaceが実行される独立した処理単位です。クラウドベースやオンプレミス環境を含む、さまざまなインフラストラクチャ上で動作できます。Targetは必要な計算リソースを提供し、定義されたインフラストラクチャ内でセキュリティ要件を満たします。

1. 以下のコマンドを実行してTargetを作成します：

```shell
daytona target create
```

2. Targetに使用する[Target Configuration](/docs/ja/configuration/target-config)を選択します。

3. デプロイしたい環境の[Provider](/docs/ja/configuration/providers)を選択します。

4. Targetの名前を入力します。

5. プロンプトが表示されたら、適切な設定オプションを入力します。設定オプションは選択したProviderによって異なります。

```text
✓ Request submitted
Creating target <target_name>
```

異なる開発環境用に追加の[Targets](/docs/ja/usage/targets)を設定することで、必要に応じてそれらを管理し、切り替えることができます。

`daytona target --help`コマンドを使用して、Targetを設定するための利用可能な設定オプションを確認してください。

## Workspaceの作成

開発環境を管理するためにDaytonaを使用して[Workspaceを作成](/docs/ja/usage/workspaces#create-a-workspace)します。Workspaceは、コードの管理、プロジェクトのビルド、チームとの連携を行うことができる独立した開発環境を表します。

Workspaceを作成するには、`daytona create`コマンドを使用し、プロンプトに従ってWorkspace設定を構成します。作成後、Workspaceにアクセスしてコードベースと対話できます。

1. 次のコマンドを実行して、ガイド付きWorkspace作成プロセスを開始します：

```shell
daytona create
```

2. 提供される3つのオプションのうち1つを選択します：

- **リポジトリのリストから選択**
- **カスタムリポジトリURLを入力**
- **サンプルから作成**

3. リストからGitリポジトリを選択するか、WorkspaceにリンクしたいGitリポジトリのURLを入力するか、サンプルプロジェクトからWorkspaceを作成することを選択します。

4. Workspaceの名前を入力します。

5. Workspaceをデプロイする[Target](/docs/ja/usage/targets)を選択します。Targetを選択すると、[Target Configuration](/docs/ja/configuration/target-config)の入力を求められます。設定は選択したTargetによって異なります。

Targetの選択と設定が完了すると、Daytonaは指定されたGitリポジトリをクローンし、その内容に基づいて新しいWorkspaceをセットアップします。ターミナルにWorkspace作成プロセスの進行状況が表示されます。

```text
<Target>    | ✓ Request submitted
<Workspace> | Creating workspace <workspace> (<workspace_id>)
<Workspace> | Pulling image...
<Workspace> | Pulling from daytonaio/workspace-project
<Workspace> | Digest: sha256:<sha256>
<Workspace> | Status: Image is up to date for daytonaio/workspace-project:latest
<Workspace> | ✓ Image pulled successfully
<Workspace> | UIDs and GIDs are the same (1000:1000).
<Workspace> | Cloning into '/workdir/<workspace>'...
<Workspace> | UIDs and GIDs are the same (1000:1000).
<Workspace> | ✓ Workspace <workspace> created
```

`daytona create --help`コマンドを使用して、Workspace作成時に利用可能な設定オプションを確認できます。

## Workspaceとの対話

[Agent Toolbox](/docs/ja/usage/agent-toolbox)を使用すると、Workspace環境内でさまざまな操作を実行するためのWorkspace Toolbox APIを提供することで、プログラム的にWorkspaceと対話できます。

Workspace Toolbox APIは、ファイルシステム、Git、およびプロセス操作を提供し、Workspace内でファイルの管理、Gitコマンドの実行、プロセスの実行を行うことができます。
