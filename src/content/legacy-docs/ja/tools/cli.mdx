`daytona`コマンドラインツールは、Daytonaのコア機能へのアクセスを提供します。
`daytona`ツールは以下の操作に使用できます：

* Daytona Serverのライフサイクル管理。
* [Workspaces](../usage/workspaces)、[Git Providers](../configuration/git-providers)、[Providers](../configuration/providers)、およびその他のDaytonaコンポーネントの管理。
* [Daytona Server](../configuration/server)のインタラクティブな設定。

このリファレンスでは、`daytona`コマンドラインツールでサポートされているすべてのコマンドを、その動作の説明とサポートされているフラグと共に一覧表示しています。
`daytona`を実行する際に`--help`/`-h`フラグを追加することで、コマンドごとにこのドキュメントにアクセスできます。

<Aside type="note">
このリファレンスは、Workspace内で実行される`daytona`コマンドには適用されません。
</Aside>

## daytona create

ワークスペースを作成する

```shell
daytona create [REPOSITORY_URL | WORKSPACE_CONFIG_NAME]... [flags]
```

__フラグ__
| Long | Short | Description |
| :--- | :---- | :---------- |
| `--blank` |  | 既存のテンプレートを使用せずに空のワークスペースを作成する |
| `--branch` |  | ワークスペースで使用するGitブランチを指定する |
| `--builder` |  | ビルダーを指定する（現在はauto/devcontainer/none） |
| `--custom-image` |  | フラグ値として渡されたカスタムイメージでワークスペースを作成する；--custom-image-userフラグも設定する必要があります |
| `--custom-image-user` |  | フラグ値として渡されたカスタムイメージユーザーでワークスペースを作成する；--custom-imageフラグも設定する必要があります |
| `--devcontainer-path` |  | フラグ値として渡されたパスでdevcontainerビルダーを自動的に割り当てる |
| `--env` |  | 環境変数を指定する（例：--env 'KEY1=VALUE1' --env 'KEY2=VALUE2' ...） |
| `--git-provider-config` |  | GitプロバイダーのコンフィギュレーションIDまたはエイリアスを指定する |
| `--ide` | `-i` | IDEを指定する（vscode, code-insiders, browser, cursor, codium, codium-insiders, ssh, jupyter, fleet, positron, zed, windsurf, clion, goland, intellij, phpstorm, pycharm, rider, rubymine, webstorm） |
| `--label` |  | ラベルを指定する（例：--label 'label.key1=VALUE1' --label 'label.key2=VALUE2' ...） |
| `--manual` |  | Gitリポジトリを手動で入力する |
| `--multi-workspace` |  | 複数のワークスペース/リポジトリをターゲットにする |
| `--no-ide` | `-n` | ターゲット作成後にIDEでターゲットを開かない |
| `--target` | `-t` | ターゲットを指定する（例：'local'） |
| `--yes` | `-y` | すべてのプロンプトを自動的に確認する |
| `--help` |  | daytonaのヘルプ |
