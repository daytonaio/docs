## Sandbox

```python
class Sandbox(SandboxDto)
```

Daytona Sandbox を表します。

**属性**:

- `fs` _FileSystem_ - ファイルシステム操作インターフェース。
- `git` _Git_ - Git 操作インターフェース。
- `process` _Process_ - プロセス実行インターフェース。
- `id` _str_ - Sandbox の一意な識別子。
- `organization_id` _str_ - Sandbox の組織ID。
- `snapshot` _str_ - Sandbox の作成に使用された Daytona スナップショット。
- `user` _str_ - Sandbox 内で実行されている OS ユーザー。
- `env` _Dict[str, str]_ - Sandbox 内で設定された環境変数。
- `labels` _Dict[str, str]_ - Sandbox に付与されたカスタムラベル。
- `public` _bool_ - Sandbox が公開アクセス可能かどうか。
- `target` _str_ - Sandbox が実行されるランナーのターゲット場所。
- `cpu` _int_ - Sandbox に割り当てられた CPU 数。
- `gpu` _int_ - Sandbox に割り当てられた GPU 数。
- `memory` _int_ - Sandbox に割り当てられたメモリ量（GiB）。
- `disk` _int_ - Sandbox に割り当てられたディスク容量（GiB）。
- `state` _SandboxState_ - Sandbox の現在の状態（例: "started", "stopped"）。
- `error_reason` _str_ - Sandbox がエラー状態の場合のエラーメッセージ。
- `backup_state` _SandboxBackupStateEnum_ - Sandbox バックアップの現在の状態。
- `backup_created_at` _str_ - バックアップが作成された日時。
- `auto_stop_interval` _int_ - 自動停止までの間隔（分）。
- `auto_archive_interval` _int_ - 自動アーカイブまでの間隔（分）。
- `runner_domain` _str_ - Sandbox ランナーのドメイン名。
- `volumes` _List[str]_ - Sandbox にアタッチされたボリューム。
- `build_info` _str_ - ダイナミックビルドから作成された場合の Sandbox のビルド情報。
- `created_at` _str_ - Sandbox が作成された日時。
- `updated_at` _str_ - Sandbox が最後に更新された日時。

#### Sandbox.create\_lsp\_server

```python
def create_lsp_server(language_id: LspLanguageId,
                      path_to_project: str) -> LspServer
```

新しい Language Server Protocol (LSP) サーバーインスタンスを作成します。

LSP サーバーは、コード補完や診断などの言語固有の機能を提供します。

**引数**:

- `language_id` _LspLanguageId_ - 言語サーバーの種類（例: LspLanguageId.PYTHON）。
- `path_to_project` _str_ - プロジェクトのルートディレクトリへのパス。相対パスはユーザーの
  ルートディレクトリを基準に解決されます。
  
**戻り値**:

- `LspServer` - 指定した言語用に設定された新しい LSP サーバーインスタンス。
  
**例**:

```python
lsp = sandbox.create_lsp_server("python", "workspace/project")
```

#### Sandbox.set\_labels

```python
@intercept_errors(message_prefix="Failed to set labels: ")
def set_labels(labels: Dict[str, str]) -> Dict[str, str]
```

Sandbox のラベルを設定します。

ラベルは、Sandbox を整理し識別するために使用できるキーと値のペアです。

**引数**:

- `labels` _Dict[str, str]_ - Sandbox のラベルを表すキーと値のペアの辞書。
  
**戻り値**:

  Dict[str, str]: 更新された Sandbox のラベルを含む辞書。
  
**例**:

```python

    "project": "my-project",
    "environment": "development",
    "team": "backend"
})
print(f"Updated labels: {new_labels}")
```

#### Sandbox.start

```python
@intercept_errors(message_prefix="Failed to start sandbox: ")
@with_timeout(error_message=lambda self, timeout: (
    f"Sandbox {self.id} failed to start within the {timeout} seconds timeout period"
))
def start(timeout: Optional[float] = 60)
```

Sandbox を起動し、準備が整うまで待機します。

**引数**:

- `timeout` _Optional[float]_ - 待機する最大時間（秒単位）。0 はタイムアウトなしを意味します。デフォルトは 60 秒です。
  
**例外**:

- `DaytonaError` - timeout が負の値の場合。Sandbox の起動に失敗した場合やタイムアウトした場合。
  
**例**:

```python
sandbox = daytona.get_current_sandbox("my-sandbox")
sandbox.start(timeout=40)  # 最大 40 秒待機
print("Sandbox started successfully")
```

#### Sandbox.wait\_for\_sandbox\_start

```python
@intercept_errors(
    message_prefix="Failure during waiting for sandbox to start: ")
@with_timeout(error_message=lambda self, timeout: (
    f"Sandbox {self.id} failed to become ready within the {timeout} seconds timeout period"
))
def wait_for_sandbox_start(timeout: Optional[float] = 60) -> None
```

Sandbox が「started」状態になるまで待機します。Sandbox の状態をポーリングし、「started」状態になるか、エラーが発生するか、タイムアウトするまで待機します。

**引数**:

- `timeout` _Optional[float]_ - 待機する最大時間（秒単位）。0 はタイムアウトなしを意味します。デフォルトは 60 秒です。
  
**例外**:

- `DaytonaError` - timeout が負の値の場合；Sandbox の起動に失敗した場合やタイムアウトした場合

#### Sandbox.wait\_for\_sandbox\_stop

```python
@intercept_errors(
    message_prefix="Failure during waiting for sandbox to stop: ")
@with_timeout(error_message=lambda self, timeout: (
    f"Sandbox {self.id} failed to become stopped within the {timeout} seconds timeout period"
))
def wait_for_sandbox_stop(timeout: Optional[float] = 60) -> None
```

Sandbox が「stopped」状態になるまで待機します。Sandbox の状態をポーリングし、「stopped」状態になるか、エラーが発生するか、タイムアウトするまで待機します。Sandbox の停止を最大 60 秒間待機します。

**引数**:

- `timeout` _Optional[float]_ - 待機する最大時間（秒単位）。0 はタイムアウトなしを意味します。デフォルトは 60 秒です。
  
**例外**:

- `DaytonaError` - timeout が負の値の場合。Sandbox の停止に失敗した場合やタイムアウトした場合。

#### Sandbox.set\_autostop\_interval

```python
@intercept_errors(message_prefix="Failed to set auto-stop interval: ")
def set_autostop_interval(interval: int) -> None
```

Sandbox の自動停止間隔を設定します。

指定した間隔の間アイドル状態（新しいイベントがない状態）が続くと、Sandbox は自動的に停止します。イベントには、SDK を通じた Sandbox への状態変更や操作が含まれます。Sandbox Previews を使った操作は含まれません。

**引数**:

- `interval` _int_ - 自動停止までの非アクティブ時間（分単位）。0 に設定すると自動停止を無効化します。デフォルトは 15 分です。
  
**例外**:

- `DaytonaError` - interval が負の値の場合
  
**例**:

```python
# Auto-stop after 1 hour
sandbox.set_autostop_interval(60)
# Or disable auto-stop
sandbox.set_autostop_interval(0)
```

#### Sandbox.set\_auto\_archive\_interval

```python
@intercept_errors(message_prefix="Failed to set auto-archive interval: ")
def set_auto_archive_interval(interval: int) -> None
```

Sandboxの自動アーカイブ間隔を設定します。

Sandboxが指定した間隔、連続して停止状態のままである場合、自動的にアーカイブされます。

**引数**:

- `interval` _int_ - 連続して停止状態のSandboxが自動アーカイブされるまでの分数。
  0を指定すると最大間隔になります。デフォルトは7日間です。
  
**例外**:

- `DaytonaError` - intervalが負の値の場合
  
**例**:

```python
# 1時間後に自動アーカイブ
sandbox.set_autoarchive_interval(60)
# または最大間隔を使用
sandbox.set_autoarchive_interval(0)
```

#### Sandbox.get\_preview\_link

```python
@intercept_errors(message_prefix="Failed to get preview link: ")
def get_preview_link(port: int) -> PortPreviewUrl
```

指定したポートでsandboxのプレビューリンクを取得します。ポートが閉じている場合は、
自動的に開かれます。プライベートなsandboxの場合、URLへのアクセス権を付与する
トークンが含まれます。

**引数**:

- `port` _int_ - プレビューリンクを開くポート番号。
  
**戻り値**:

- `PortPreviewUrl` - プレビューリンクのレスポンスオブジェクト。`url`と
  （プライベートsandboxにアクセスするための）`token`が含まれます。
  
**例**:

```python
preview_link = sandbox.get_preview_link(3000)
print(f"Preview URL: {preview_link.url}")
print(f"Token: {preview_link.token}")
```

#### Sandbox.archive

```python
@intercept_errors(message_prefix="Failed to archive sandbox: ")
def archive() -> None
```

sandboxをアーカイブし、非アクティブ化してその状態を保存します。sandboxが
アーカイブされると、ファイルシステム全体の状態がコスト効率の高いオブジェクトストレージに
移動され、長期間sandboxを利用可能な状態に保つことができます。アーカイブ状態と
停止状態の違いは、アーカイブされたsandboxの起動にはサイズに応じてより多くの
時間がかかる点です。アーカイブする前にsandboxを停止しておく必要があります。

## リソース

```python
@dataclass
class Resources()
```

Sandbox のためのリソース設定。

**属性**:

- `cpu` _Optional[int]_ - 割り当てる CPU コア数。
- `memory` _Optional[int]_ - 割り当てるメモリ容量（GiB）。
- `disk` _Optional[int]_ - 割り当てるディスク容量（GiB）。
- `gpu` _Optional[int]_ - 割り当てる GPU 数。
  
**例**:

```python
resources = Resources(
    cpu=2,
    memory=4,  # 4GiB RAM
    disk=20,   # 20GiB disk
    gpu=1
)
params = CreateSandboxFromImageParams(
    image=Image.debian_slim("3.12").pip_install(["numpy", "pandas"]),
    language="python",
    resources=resources
)
```
