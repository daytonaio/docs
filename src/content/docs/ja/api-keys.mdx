Daytona API keyは[Daytona API](/docs/ja/tools/api)へのリクエストを認証するために使用されます。

[設定](/docs/ja/configuration)ページでは、`DAYTONA_API_KEY`環境変数を含む環境変数の設定方法について説明しています。

既存のAPI Keyを一覧表示するには、DashboardのKeysサブページに移動してください。`Create Key`をクリックすると、新しいキーを作成してクリップボードにコピーするためのモーダルが表示されます。

## 有効期限

新しいAPI Keyを作成する際、オプションでキーの有効期限を設定できます。有効期限が設定されていない場合、キーは永続的に有効です。

## Permissions

新しいAPI Keyを追加する際、そのキーに対する権限セットの選択を求められます：

| Permission | Description |
|------------|-------------|
| **`Sandboxes`** | Sandboxesの作成と管理を行う権限を付与します |
| **`Snapshots`** | Snapshotsの作成と管理を行う権限を付与します |
| **`Registries`** | Registriesの作成と管理を行う権限を付与します |

これらの権限にはそれぞれ、リソースの作成と更新（例：`write:sandboxes`）およびリソースの削除（例：`delete:sandboxes`）のスコープがあり、個別に付与することができます。
